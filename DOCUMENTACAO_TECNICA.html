Documentação Técnica

Guia completo de arquitetura, deploy e manutenção do sistema VendaCerta.

1. Visão Geral do Sistema

O VendaCerta é uma plataforma robusta de gestão de vendas e comissões, desenvolvida para oferecer alta performance, escalabilidade e segurança. O sistema gerencia vendedores, metas, clientes, produtos e ordens de serviço, com um painel administrativo completo e dashboards responsivos.

Stack Tecnológico
Python 3.11 Flask 3.0 PostgreSQL 16 Bootstrap 5.3 Railway Gunicorn
Métricas do Projeto
  • Arquivos Python: 25+
  • Linhas de Código: ~15.000
  • Modelos de Dados: 16 tabelas
  • Rotas/Endpoints: 200+
  • Cobertura de Testes: Parcial

2. Infraestrutura e Deploy (Railway)

O sistema foi otimizado especificamente para a plataforma Railway, utilizando o builder Nixpacks para máxima performance.
Configuração de Build (nixpacks.toml)

Otimizações aplicadas para reduzir tempo de build e startup:

  • Cache: --no-cache-dir para evitar bloat.
  • Init: Inicialização do DB movida para fase de build.
  • Preload: Gunicorn configurado com --preload.
[phases.build]
cmds = ["python init_railway.py"]

[start]
cmd = "gunicorn app:app --preload"
Monitoramento e Health Check

Endpoint dedicado para verificação de saúde do serviço:

  • Rota: /ping ou /health
  • Timeout: Aumentado para 100s (railway.json).
  • Resposta: JSON ultrarrápido {"status": "ok"}.
Exemplo de resposta: 
{
  "status": "ok",
  "database": "connected"
}

3. Banco de Dados e Backups

Arquitetura Híbrida

O sistema suporta dois ambientes de banco de dados de forma transparente:

Ambiente Tecnologia Estratégia de Backup
Desenvolvimento SQLite Automático via backup_helper.py (arquivos .db locais)
Produção (Railway) PostgreSQL Nativo via Railway CLI (Snapshot diário)
Módulo backup_helper.py

Este módulo foi refatorado para ser inteligente quanto ao ambiente:

  • Detecta automaticamente se está rodando no Railway.
  • Bloqueia operações de arquivo (copy/delete) em produção para evitar erros.
  • Fornece instruções claras de uso da CLI do Railway quando invocado em produção.

4. Histórico de Correções Críticas

1. Fix de Deploy Railway (IndentationError)

Problema: Erro de indentação na linha 586 do app.py impedia o start da aplicação.
Solução: Remoção de código duplicado e correção da sintaxe Python.

2. Otimização de Timeout e Health Check

Problema: Deploy falhava por timeout na verificação de saúde.
Solução: Criação de rota /ping leve, aumento do timeout para 100s e ajuste no railway.json.

3. Loop de Redirecionamento HTTPS

Problema: O Health Check do Railway recebia 301 (Redirect) ao invés de 200.
Solução: Exceção adicionada no middleware force_https para ignorar rotas de monitoramento.
